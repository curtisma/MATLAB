Examples

Once 'Toolbox Tools' is installed, run the following command in the MATLAB Command Window:

mksandbox( 'mytoolbox' )

This creates a sandbox for a MATLAB toolbox named 'mytoolbox'. You can also specify the sandbox location as follows:

mksandbox( 'mytoolbox', 'C:\Work\Toolboxes' )

After the sandbox has been created, run release( 'mytoolbox' ) to package the toolbox.

Customising sandbox content

The sandbox created contains a number of MATLAB files to get you started on building your own toolbox. You can customise the content as described below.

Project file

The project file mytoolbox.prj contains information about the toolbox and is used for packaging the .mltbx file. Specify toolbox metadata, i.e. author information, toolbox summary, description, and image, code, documentation, apps, and examples to be included in the packaged toolbox.

Shipping toolbox files

The /tbx folder contains code and documentation to be released to end users. The /tbx/mytoolbox folder contains the toolbox code, i.e. MyClass.m and myfunc.m, and a Contents.m file summarising the MATLAB files in the folder.
This enables users to query content and version information of the toolbox using the help or ver commands. See Create Help Summary Files on how to create a Contents.m file.

The /tbx/doc folder contains supporting documentation or tutorials to demonstrate the toolbox functionalities. To display custom documentation in the MATLAB Help browser, the toolbox should include an info.xml and helptoc.xml file. You can also include custom examples in the toolbox and these example files should be stored under the /tbx/doc/examples folder.

Tests

The /tests folder contains tests for your toolbox files. An example test file MyTest.m is provided to demonstrate how to write Class-Based Unit Tests using the MATLAB Unit Test Framework .

The addsandbox.m and rmsandbox.m scripts

These files are used to automate the configuration of the MATLAB environment for development, packaging and testing.

When writing and packaging your toolbox, you want the sandbox code to be on your MATLAB path. The addsandbox.m script adds the /tbx folder and all its subfolders to the MATLAB path when executed. You should customise this to your own development environment.

When testing the packaged .mltbx file, it is important that the sandbox code is removed from the MATLAB path. Similarly, the rmsandbox.m script removes the /tbx folder and all its subfolders from the MATLAB path when executed. Again, you should customise this script to your development environment.

The release.m script

An example release.m script is provided to automate the toolbox release process. This script:

  • runs checks for the correct MATLAB version and dependencies,
  • generates documentation and publishing examples,
  • runs tests, and
  • packages the toolbox.

You should customise this file to your toolbox release process.

An archive of all toolbox releases

All toolbox releases should be archived in the /releases folder for ease of reference. Each toolbox release should be annotated with the correct version number. This is automated in the example release.m script provided.
